# MoAS 개선 명세: Class-Conditional Modality Reliability

## 1. 문제 정의

### 1.1 현상

Table 4 (Scoring Strategy Ablation) 결과에서 **Uniform Average와 Ours(Adaptive Reliability)의 성능이 사실상 동일**하다.

| Strategy | AUC_Short | AUC_Mid | AUC_Long |
|---|---|---|---|
| Main Only | 87.50 | 87.33 | 84.04 |
| Uniform Sum | 87.85 | 88.17 | 84.46 |
| Uniform Average | 88.03 | 88.37 | 84.95 |
| Adaptive Entropy | 87.72 | 88.05 | 84.71 |
| **Adaptive Reliability (Ours)** | **88.03** | **88.36** | **84.95** |

이는 MoAS가 계산한 adaptive weight α_m이 모든 샘플에 대해 거의 균등(≈1/3, 1/3, 1/3)하게 출력됨을 의미한다.

### 1.2 현재 MoAS 파이프라인

```
z_m → E_m = -log Σ_c exp(z_{m,c}) → r_m = σ(-E_m) → α_m = softmax(r_m / τ)
```

- `E_m`: modality m의 energy score
- `r_m`: sigmoid로 정규화된 reliability score (0~1)
- `α_m`: softmax로 정규화된 modality weight

최종 예측:
```
z^MoAS_c(x) = z_{main,c}(x) + Σ_m α_m(x) · z_{m,c}(x)
s(x) = max_c z^MoAS_c(x)
```

### 1.3 근본 원인

Energy 기반 reliability가 adaptive하지 않은 이유는 **4가지**로 분석된다.

| 원인 | 설명 |
|---|---|
| **"자신감 ≠ 정확도"** | Energy는 modality head가 "얼마나 확신하는가"만 측정. "얼마나 정확한가"는 반영하지 않음. 틀린 예측에도 높은 에너지(높은 확신)를 보일 수 있음. |
| **Class-agnostic** | Energy는 모든 클래스의 logit을 하나의 스칼라로 합산하므로, "이 modality가 특정 클래스에 대해 얼마나 잘 맞추는가"라는 정보가 소실됨. |
| **Modality 간 구조적 유사성** | 3개 modality head가 동일한 학습 신호(MoRST CE loss)로 학습되어 에너지 수준이 체계적으로 비슷해짐. |
| **Sigmoid 포화** | `log Σ exp(z_{m,c})`가 큰 양수이므로 sigmoid 입력이 항상 포화 영역 → r_m ≈ 1 → softmax(r/τ) ≈ (1/3, 1/3, 1/3). |

**핵심**: 진짜 필요한 것은 "이 modality가 이 샘플에 대해 정확한 예측을 하고 있는가"이지만, 현재는 "이 modality의 logit magnitude가 얼마나 큰가"만 측정하고 있다.

### 1.4 이미 시도한 해결책과 결과

| 시도 | 결과 |
|---|---|
| Sigmoid 제거 후 energy를 softmax에 직접 투입 | 성능 크게 하락 |
| Energy 정규화 후 sigmoid | 근본적 해결 불가 (energy 자체가 reliability의 적절한 지표가 아님) |
| Max softmax probability | 근본적 해결 불가 |

## 2. 개선 방향: Class-Conditional Modality Reliability

### 2.1 핵심 아이디어

Replay memory buffer의 exemplar를 활용하여 **클래스별 modality의 실제 정확도(accuracy)**를 사전에 계산하고, inference 시 예측 클래스에 따라 modality 가중치를 adaptive하게 결정한다.

### 2.2 구체적 설계 (방향 A: Class-Conditional Accuracy Prior)

#### Step 1: Reliability Matrix 구축 (각 task 학습 완료 후)

Buffer `B`의 모든 exemplar `(x, y)`에 대해 각 modality head의 클래스별 정확도를 계산한다.

```python
# R: reliability matrix, shape = (num_modalities, num_classes)
# B_c: buffer 내 class c에 속하는 샘플 집합

for m in modalities:          # {rgb, gyro, acce}
    for c in known_classes:
        z_m = H_m(F_m(x_m))   # modality-specific logit
        y_hat_m = argmax(z_m)
        R[m, c] = count(y_hat_m == c for x in B_c) / |B_c|
```

#### Step 2: Inference 시 adaptive weight 계산

```python
# 기존과 동일하게 z_main, z_m 계산
z_main = C(G({F_m(x_m)}))
z_m = H_m(F_m(x_m))      # for each m

# z_main에서 예측 확률 분포
p = softmax(z_main)       # shape = (num_classes,)

# 각 modality의 class-conditional reliability를 예측 분포로 가중 평균
for m in modalities:
    w[m] = sum(p[c] * R[m, c] for c in known_classes)

# modality weight
alpha = softmax(w / tau)   # shape = (num_modalities,)

# MoAS-integrated logit (기존과 동일한 구조)
z_MoAS[c] = z_main[c] + sum(alpha[m] * z_m[c] for m in modalities)

# novelty score
s = max(z_MoAS)
```

#### 수식 정리

**Reliability matrix** (task t 학습 완료 후 1회 계산):
```
R_{m,c} = |{x ∈ B_c : argmax H_m(F_m(x_m)) = c}| / |B_c|
```

**Inference-time modality weight**:
```
p(c|x) = softmax(z_main(x))
w_m(x) = Σ_c p(c|x) · R_{m,c}
α_m(x) = exp(w_m / τ) / Σ_j exp(w_j / τ)
```

**최종 logit 및 novelty score** (기존과 동일):
```
z^MoAS_c(x) = z_{main,c}(x) + Σ_m α_m(x) · z_{m,c}(x)
s(x) = max_c z^MoAS_c(x)
```

### 2.3 기존 대비 변경 요약

| 항목 | 기존 (Energy 기반) | 개선 (Class-Conditional) |
|---|---|---|
| Reliability 지표 | Energy score (sample-level, class-agnostic) | Class-conditional accuracy (class-level, accuracy 기반) |
| Sigmoid | 필요 (energy 정규화 용도) | 불필요 (R은 이미 [0,1] 범위) |
| Energy 계산 | 필요 | 불필요 |
| 추가 데이터 | 없음 | Replay buffer (이미 MoRST에서 사용 중) |
| 추가 계산 | 없음 | Task 학습 완료 시 R matrix 1회 계산 |
| Sample-adaptivity | Energy 차이에 의존 (거의 없음) | 예측 클래스 p(c\|x)에 따라 달라짐 |

## 3. 확장 방향 (방향 A 이후 시도 가능)

### 3.1 방향 B: 정적 + 동적 결합 (Hybrid)

방향 A의 class-conditional prior에 sample-level 동적 신뢰도를 곱한다.

```python
# 동적: 현재 샘플에 대한 modality 수준 confidence
q_m = max(softmax(z_m))

# 결합
w_m = q_m * sum(p[c] * R[m, c] for c in classes)
alpha = softmax(w / tau)
```

목적: RGB가 클래스 A에 보통은 강하지만, 특정 샘플이 흐릿한 영상이면 동적 신뢰도가 낮아져 보정.

### 3.2 방향 C: Modality-Main Agreement Score

GT label 대신 main classifier 예측과의 일치도를 신뢰도로 사용.

```python
A[m, c] = mean(argmax(z_m) == argmax(z_main) for x in B_c)
```

main classifier를 pseudo-reference로 사용하여 GT label 의존도를 줄임.

## 4. 구현 시 주의사항

### 4.1 R matrix 갱신 타이밍

- 각 **task 학습 완료 시점**에 R을 재계산
- 이 시점에 buffer에 저장된 exemplar의 modality logit은 이미 최신 모델로 계산 가능
- R은 task가 진행될수록 크기가 커짐: `(|M|, |C_t|)` where `C_t`는 task t까지의 누적 클래스 수

### 4.2 Buffer 크기와 안정성

- Buffer 내 클래스별 샘플 수가 적으면 R 추정이 불안정
- 해결: Laplace smoothing 적용 가능
  ```python
  R[m, c] = (correct_count + epsilon) / (total_count + epsilon * num_classes)
  ```

### 4.3 Temperature τ 설정

- R 값의 범위가 [0, 1]이고 modality 간 차이가 작을 수 있으므로, τ를 낮게 설정하여 차이를 증폭해야 할 수 있음
- Hyperparameter search 범위 제안: τ ∈ {0.01, 0.05, 0.1, 0.5, 1.0}

### 4.4 신규 클래스 처리

- Task t에서 새로 등장한 클래스는 아직 R에 없음
- 새 클래스에 대해서는 학습 직후 buffer에 추가된 exemplar로 즉시 R 계산
- 또는 새 클래스의 R을 uniform(1/|M|)으로 초기화 후, 다음 task 완료 시 갱신

## 5. 검증 계획

| 검증 항목 | 방법 | 기대 결과 |
|---|---|---|
| R matrix 분산 | R_{m,c} 값을 heatmap으로 시각화 | Modality와 class에 따라 유의미한 차이 존재 |
| α 분포 | 전체 테스트 샘플의 α_m 히스토그램 | Uniform (1/3, 1/3, 1/3)과 유의미하게 다른 분산 |
| Known vs Novel α 차이 | Known/Novel 샘플 그룹별 α 분포 비교 | 두 그룹에서 다른 weight 패턴 |
| Uniform Average 대비 성능 갭 | AUC, FPR95 비교 | 통계적으로 유의미한 차이 |
| Buffer 크기 민감도 | Buffer 크기 변경 실험 | 작은 buffer에서도 안정적 |
