## 8. $E_{\text{uniform}}$ 보정 적용 시 예상 효과

> **이 섹션은 Section 9(에너지 스케일 진단) 기반으로 도출된 분석이다.**  
> 보정식 $r_m = \sigma(-E_m + \ln C)$를 현재 측정 에너지에 수치적으로 적용하여 α_m 분포 변화를 예측한다.

### 8.1 보정식

$$r_m^{\text{corrected}} = \sigma\!\bigl(-(E_m - E_{\text{uniform}})\bigr) = \sigma(-E_m + \ln C)$$

- **현재 구현**: $r_m = \sigma(-E_m)$ — sigmoid 0점이 $E_m = 0$으로 고정 (설계 결함)
- **보정 후**: $E_{\text{uniform}} = -\ln C$를 0점으로 이동 → 균등 분포 시 $r_m = 0.5$ (중립)

## 9. 추가 진단: Modality별 Energy Scale 분석

**실험 일시**: 2026-03-09 (Step 1 보완)  
**목적**: energy 포화의 구체적 원인 — modality별 energy scale이 $E_{\text{uniform}} = -\ln(C)$ 대비 어떻게 분포하는지 확인

### 9.1 결과 데이터

기준값: 균등 분포 가정 시 $E_{\text{uniform}} = -\ln(C)$ (모든 클래스에 동일 확률이면 이 값)

| Task | C | $E_{\text{uniform}}$ | RGB E mean | Gyro E mean | Acce E mean | RGB 포화% | Gyro 포화% | Acce 포화% |
|------|---|---------------------|-----------|------------|------------|----------|-----------|-----------|
| 0 | 8  | -2.079 | -2.729 | -5.280 | -4.227 | 62.5% | **100%** | **100%** |
| 1 | 16 | -2.773 | -2.153 | -5.418 | -5.291 | 50.0% | **100%** | **100%** |
| 2 | 24 | -3.178 | -1.223 | -5.698 | -6.197 | 12.5% | **100%** | **100%** |
| 3 | 32 | -3.466 | -0.488 | -6.112 | -6.195 | **0.0%** | **100%** | **100%** |

포화 기준: $E_m < E_{\text{uniform}}$ → sigmoid(-E) > sigmoid($-E_{\text{uniform}}$) → 과도하게 높은 confidence

### 9.2 두 가지 독립적 포화 원인 발견

#### 원인 ①: Gyro/Acce — 절대적 포화 (모든 Task에서 100%)

$$E_{\text{Gyro}} \approx -5.3 \sim -6.1 \quad \ll \quad E_{\text{uniform}} = -2.1 \sim -3.5$$

- Gyro/Acce의 energy가 $E_{\text{uniform}}$보다 **2.5~3.0 단위 낮음**
- 즉 IMU 모달리티는 모든 클래스에 대해 과확신(overconfident) 상태
- `sigmoid(-(-6)) = sigmoid(6) ≈ 0.9975` → 완전 포화
- **원인**: IMU logit의 절댓값 magnitude 자체가 RGB보다 크게 학습됨

#### 원인 ②: RGB — 구조적 문제 (클래스 수 증가로 악화)

| Task | RGB E mean | $E_{\text{uniform}}$ | 차이 ($E - E_{\text{uniform}}$) |
|------|-----------|---------------------|-------------------------------|
| 0 | -2.729 | -2.079 | -0.650 (E_uniform보다 낮음, 약간 과확신) |
| 1 | -2.153 | -2.773 | +0.620 (E_uniform보다 높음, 불확실) |
| 2 | -1.223 | -3.178 | +1.955 (E_uniform보다 훨씬 높음, 매우 불확실) |
| 3 | -0.488 | -3.466 | +2.978 (E_uniform보다 3.0 높음, 극도로 불확실) |

- 클래스 수가 늘어날수록 RGB는 점점 더 불확실해짐 (energy 상승)
- 반면 Gyro/Acce는 클래스가 늘어도 energy가 더 낮아짐 (포화 심화)
- 결과: **softmax([r_RGB, r_Gyro, r_Acce])에서 RGB 비율이 오히려 커질 수도 있지만, 세 값이 모두 sigmoid로 압축되어 실질적 차이가 소멸**

### 9.3 설계 결함 진단

현재 구현:
$$r_m = \sigma(-E_m)$$

문제: sigmoid는 **절댓값 기준**으로 변환 → 클래스 수에 무관하게 0.5 기준점 고정  
그러나 클래스 수가 늘면 $E_{\text{uniform}} = -\ln(C)$도 함께 낮아지므로,  
**올바른 0점 기준은 $E_{\text{uniform}}$이어야 함**:

$$r_m^{\text{corrected}} = \sigma(-(E_m - E_{\text{uniform}})) = \sigma(-E_m - \ln C)$$

| 상태 | $E_m$ 위치 | 현재 $\sigma(-E_m)$ | 보정 후 $\sigma(-E_m - \ln C)$ |
|------|-----------|---------------------|-------------------------------|
| 완전 균등분포 | $E_m = E_{\text{uniform}}$ | $\sigma(\ln C) > 0.5$ (**편향**) | **0.5** (중립) |
| ID (confident) | $E_m \ll E_{\text{uniform}}$ | ≈ 1.0 (포화) | > 0.5 (적절) |
| OOD (uncertain) | $E_m \gg E_{\text{uniform}}$ | 낮음 | < 0.5 (적절) |

### 9.4 결론 및 개선 방향 도출

**포화의 근본 원인 2가지**:
1. **Sigmoid 절대 기준 문제**: $-\ln(C)$를 0점으로 두지 않아 클래스 수 증가 시 왜곡
2. **IMU logit magnitude 과대**: Gyro/Acce가 RGB보다 logit 크기가 구조적으로 크게 학습됨

**즉각적 개선 옵션 (방향 D 재정의)**:
```python
# 현재 (문제)
confidence = torch.sigmoid(-energy)

# 개선안 1: E_uniform 기준 보정
num_classes = logits.shape[1]
energy_uniform = -torch.log(torch.tensor(num_classes, dtype=torch.float32))
confidence = torch.sigmoid(-(energy - energy_uniform))  # = sigmoid(-energy + ln(C))

# 개선안 2: Temperature scaling으로 분산 확대
temperature = 0.5  # < 1 이면 sigmoid 기울기 증가
confidence = torch.sigmoid(-energy / temperature)
```

**개선안 1이 이론적으로 올바른 이유**:
- 균등 분포 시 confidence = 0.5 (neutral)
- ID 데이터 시 confidence > 0.5 (confident)  
- OOD 데이터 시 confidence < 0.5 (uncertain)
- 클래스 수 변화에 robust (자동 보정)

### 11.7 Z-score 해석의 전환: Absolute Confidence에서 Relative Atypicality로

#### 11.7.1 해석의 변화

기존 `sigmoid(-E)` 또는 `sigmoid(-(E - E_uniform))` 방식은 energy를 **절대적 confidence / density proxy**로 해석한다.
즉, "이 modality의 예측이 얼마나 확신이 높은가"를 측정하는 것이다.

Z-score 정규화는 이 해석을 근본적으로 전환한다:

$$z_m = \frac{E_m - \mu_{m,\text{mem}}}{\sigma_{m,\text{mem}}} \quad\Rightarrow\quad \text{confidence}_m = -z_m$$

이제 confidence는 **"이 샘플의 energy가 해당 modality의 학습 분포에서 얼마나 비전형적(atypical)인가"**를 측정한다.  
이는 raw energy의 물리적 의미 일부를 버리는 대신, **cross-modality comparability (모달리티 간 비교 가능성)**을 얻는 trade-off이다.

| 관점 | 기존 (raw / E_uniform) | Z-score |
|------|----------------------|--------|
| 측정 대상 | 절대적 확신도 (absolute confidence) | 모달리티 내 상대적 비전형성 (relative atypicality) |
| 0점 기준 | $E=0$ 또는 $E=E_{\text{uniform}}$ | $E = \mu_{m,\text{mem}}$ |
| Cross-modality scale | ✗ 비교 불가 (IMU >> RGB) | ✅ 정규화로 비교 가능 |
| 물리적 energy 해석 | 보존 | 일부 포기 |
| OOD 탐지 의미 | "이 입력이 얼마나 확실한가" | "이 입력이 memory 분포에서 얼마나 벗어나는가" |

#### 11.7.2 Cross-Modality Scale Sensitivity 완화

Z-score의 핵심 장점은 **cross-modality scale sensitivity를 완화(mitigate)**한다는 점이다.

Section 9 진단에서 확인된 구조적 문제:
- Gyro/Acce: $E \approx -5 \sim -6$ → RGB보다 2.5~3.0 단위 낮음 (IMU logit magnitude 과대)
- 어떤 보정식을 적용해도, raw energy 스케일이 다르면 sigmoid/softmax 후 한쪽이 지배

Z-score는 각 modality를 **자체 분포의 표준편차 단위로 정규화**하므로, RGB의 $z=1$과 Gyro의 $z=1$이 동일한 의미를 갖게 된다.  
실제 결과에서도 확인됨: RGB(0.58), Gyro(0.54), Acce(0.57) → 3개 modality가 거의 같은 스케일로 정규화 (Section 11.3).

#### 11.7.3 Modality-Aware Gating으로의 재해석

Z-score 기반 confidence를 "modality-aware gating" 관점에서 재해석할 수 있다:

> **"이 샘플에서 어떤 modality를 더 믿을 것인가"**

이 질문에 답하려면, 각 modality의 confidence가 **동일한 척도(comparable scale)**로 표현되어야 한다.  
Z-score confidence $= -z_m$은 정확히 이를 제공한다:

$$\alpha_m = \text{softmax}(-z_{\text{RGB}},\; -z_{\text{Gyro}},\; -z_{\text{Acce}})$$

- $-z_m$이 클수록 = 해당 modality의 energy가 memory 평균보다 낮음 = 해당 modality에서 더 확신이 높은 방향
- Modality 간 비교가 공정하게 이루어짐 (cross-modal comparable relative score)

이는 raw energy의 **절대적 확신도**로 gating하는 것과 질적으로 다른 접근이다:  
- Raw/E_uniform: "RGB가 0.87인데 Gyro도 0.99이니 비슷하다" (포화로 구별 불가)
- Z-score: "RGB는 memory 분포에서 0.5σ 아래인데, Gyro는 1.2σ 아래이다" (상대적 위치로 구별)

#### 11.7.4 Trade-off 정리

| 얻는 것 | 잃는 것 |
|--------|--------|
| Cross-modality comparability | Raw energy의 절대적 물리적 의미 |
| IMU scale sensitivity 완화 | Class 수 변화에 대한 자동 보정 ($E_{\text{uniform}}$ 없음) |
| Modality-aware gating에 적합한 상대적 스코어 | OOD 탐지 시 memory 평균 기준의 이론적 한계 (Section 11.6 확인) |
| per-sample std 0.05 달성 (Task 1, 2) | Task별 E 통계에 의존 → 통계 품질이 성능 좌우 |

> **결론**: Z-score는 α_m gating의 관점에서는 이론적으로 가장 적합한 접근이나,  
> OOD detection scoring에서는 absolute energy가 더 유리할 수 있다.  
> 따라서 **gating(α_m 계산)과 OOD scoring을 분리**하여 각각 최적의 방식을 적용하는 것이 향후 방향이 될 수 있다.
