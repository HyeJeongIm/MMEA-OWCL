## 1. 근본 원인 진단

### 1.1 왜 Energy 기반 α_m이 uniform에 수렴하는가

현재 MoAS 파이프라인:
```
E_m(x) = -log Σ_c exp(z_{m,c})          ← energy
r_m(x) = sigmoid(-E_m(x))               ← sigmoid 정규화
α_m(x) = softmax({r_m} / τ)             ← modality weight
```

**문제 ①: Sigmoid 포화**
- `log Σ exp(z_{m,c})`는 logit magnitude에 비례해 큰 양수
- sigmoid 입력이 항상 깊은 음수 영역 → `r_m ≈ 1`로 포화
- 모든 m에서 r_m ≈ 1이면 softmax 출력 → (1/3, 1/3, 1/3)

**문제 ②: Modality 간 에너지가 구조적으로 유사**
- MoRST의 동일한 CE loss 신호로 3개 head가 학습됨
- 각 head의 logit magnitude가 체계적으로 비슷해짐
- → E_RGB ≈ E_Gyro ≈ E_Acce → softmax → uniform

**문제 ③: Energy = "자신감" ≠ "정확도"**
- Energy는 "이 modality가 얼마나 확신하는가"만 측정
- "이 modality가 이 샘플에 대해 얼마나 정확한가"는 반영 안 됨
- 틀린 예측에도 높은 에너지(높은 확신) 가능

**문제 ④: Class-agnostic**
- 스칼라 하나로 전체 클래스 정보를 압축 → 클래스별 modality 강점이 소실
- 예: RGB는 동작이 큰 활동에는 강하지만 정적 활동에는 약함 → 이 정보가 없음

### 1.2 왜 Entropy 기반도 하락하는가

- Entropy 역시 softmax 정규화 과정에서 정보 손실 발생
- Entropy는 분포의 "불확실성"을 측정하지만, 세 modality head가 비슷한 불확실성 수준을 가지면 Entropy 기반 weight도 uniform으로 수렴
- 또한 Entropy는 틀린 예측에도 낮은 엔트로피(높은 확신)를 보일 수 있어 신호 오염 발생

## 2. 개선 방향 (우선순위 순)

### ★ 방향 A: Class-Conditional Modality Reliability (가장 유망)

> 이미 `moas_improvement_spec.md`에서 제안된 방향이며, 미구현 상태

**핵심 아이디어**: Energy나 Entropy 같은 sample-level 스칼라 대신,
replay buffer의 exemplar에서 **클래스별 modality 예측 정확도**를 사전에 계산한 뒤,
inference 시 main logits의 예측 분포 `p(c|x)`로 가중하여 활용

```
[Task 학습 완료 후 1회 계산]
R[m, c] = |{x ∈ B_c : argmax H_m(F_m(x)) = c}| / |B_c|   ← (|M| × |C|) 행렬

[Inference 시]
p(c|x)    = softmax(z_main(x))                             ← class 예측 분포
w_m(x)    = Σ_c p(c|x) · R[m, c]                          ← class-conditional 기대 정확도
α_m(x)    = exp(w_m / τ) / Σ_j exp(w_j / τ)               ← adaptive weight
z^MoAS_c  = z_main,c + Σ_m α_m · z_{m,c}
s(x)      = max_c z^MoAS_c
```

**왜 기존보다 나은가**:
- R[m,c]는 실제 정확도 기반이므로 "자신감 ≠ 정확도" 문제 해결
- class-conditional이므로 modality별 강점이 클래스 수준에서 반영됨
- R[m,c] ∈ [0, 1]이므로 sigmoid 포화 문제 없음
- 추가 데이터 불필요 (이미 있는 replay buffer 재사용)

**예상 α 분포의 변화**:
- 현재: α ≈ (0.33, 0.33, 0.33) (모든 샘플에 대해 동일)
- 개선 후: α가 예측 클래스와 task 진행에 따라 sample-wise로 달라짐

**주요 구현 포인트**:
```python
# task 학습 완료 후 build_rehearsal_memory() 직후에 호출
def _compute_reliability_matrix(self):
    R = np.zeros((len(self._modality), self._total_classes))
    for class_idx in range(self._total_classes):
        mask = (self._targets_memory == class_idx)
        if mask.sum() == 0:
            R[:, class_idx] = 1.0 / len(self._modality)  # uniform fallback
            continue
        # ... modality head로 예측 후 accuracy 계산
    self._reliability_matrix = R  # (|M|, |C|) numpy array

# inference 시 compute_scores_from_outputs() 내부에서
p = softmax(z_main, dim=-1)           # (B, C)
R_tensor = torch.from_numpy(self._reliability_matrix).to(device)  # (|M|, C)
w = (p.unsqueeze(1) * R_tensor.unsqueeze(0)).sum(-1)  # (B, |M|)
alpha = softmax(w / tau, dim=-1)      # (B, |M|)
```

### 방향 B: Hybrid (Static Prior + Dynamic Sample-level)

방향 A의 class-conditional prior에 sample-level 동적 신뢰도를 **곱**하여 양면 보완

```
q_m(x) = max_c softmax(z_m(x))_c         ← sample-level confidence (MSP)
w_m(x) = q_m(x) · Σ_c p(c|x) · R[m, c]  ← hybrid weight
α_m(x) = softmax(w / τ)
```

**언제 추가로 도움이 되는가**:
- RGB가 특정 클래스에 보통은 강하지만, 해당 샘플이 흐릿하거나 가려진 경우 → q_m이 낮아져 α가 자동 보정
- Prior(R)만으로는 잡기 어려운 sample-specific variation을 q_m이 보완

**우선순위**: 방향 A 구현·검증 후 추가 실험

---

### 방향 C: Modality-Main Agreement Score

GT label 대신 **main classifier와의 일치도**를 reliability로 사용 (label 독립적)

```
A[m, c] = mean({argmax z_m(x) == argmax z_main(x) | x ∈ B_c})
```

**장점**:
- GT에 의존하지 않으므로 novel class에 대한 generalizing 가능성
- main classifier를 pseudo-reference로 사용 → main과 보완적인 modality에 자연스럽게 높은 weight

**단점/위험**:
- main classifier가 이미 dominant modality에 편향돼 있으면, 그 modality에 더 높은 agreement → 편향 증폭 가능
- 먼저 방향 A로 베이스라인을 확보한 후 비교

---

### 방향 D: $E_{\text{uniform}}$ 보정 + Temperature τ 재탐색 (Quick Fix)

> **업데이트**: Step 1 에너지 스케일 진단 결과, "τ 재탐색" 외에 **sigmoid 기준점 수정**이 더 근본적인 Quick Fix임이 확인되었다.

#### D': $E_{\text{uniform}}$ 보정 (신규 추가, 우선 적용)

현재 구현의 설계 결함: $r_m = \sigma(-E_m)$ — sigmoid 0점이 $E = 0$으로 고정

**올바른 기준점**: $E_{\text{uniform}} = -\ln C$ (균등 분포 시 energy 기댓값)

$$r_m^{\text{corrected}} = \sigma\!\bigl(-(E_m - E_{\text{uniform}})\bigr) = \sigma(-E_m + \ln C)$$

```python
# auxiliary_head_fusion_v2_7.py (_compute_confidence)
energy = -torch.logsumexp(logits, dim=1)
num_classes = logits.shape[1]
energy_uniform = -torch.log(
    torch.tensor(num_classes, dtype=torch.float32, device=logits.device)
)
confidence = torch.sigmoid(-(energy - energy_uniform))
```

**이론적 보장**:
- 균등 분포 시 → confidence = 0.5 (중립)
- ID (확신) 시 → confidence > 0.5
- OOD (불확실) 시 → confidence < 0.5
- 클래스 수 증가에 자동 보정

**예상 효과**:
- Task 3 RGB: $r_{\text{RGB}}$ 0.62 → **0.05** (대폭 억제)
- Task 3 Gyro/Acce: $r_{\text{Gyro/Acce}}$ 0.998 → **~0.93** (소폭 감소)
- per-sample std(α): 0.008 → **0.15~0.20** 예상

**한계**: IMU logit magnitude 과대 문제는 여전히 존재 (Gyro/Acce 포화 완전 해소 안 됨)

#### D: Temperature τ 재탐색

$E_{\text{uniform}}$ 보정 후에도 α 차이가 작은 경우 τ를 줄여 차이를 증폭:

```
τ 탐색 범위: {0.001, 0.005, 0.01, 0.05, 0.1, 0.5, 1.0}
```

**한계**: Energy 자체가 구조적으로 균등하게 수렴하는 경우, τ 조정만으로는 근본 해결 불가  
→ $E_{\text{uniform}}$ 보정과 병행하면 시너지 가능

---

### 방향 E: Logit-Space 분리도 기반 Reliability

각 modality head logit의 **known class 간 분리도(separability)**를 reliability로 활용

```
δ_m(x) = max_c z_{m,c}(x) - second_max_c z_{m,c}(x)   ← margin (top-2 gap)
α_m(x) = softmax(δ_m / τ)
```

**직관**: margin이 클수록 해당 modality가 이 샘플에 대해 확신이 강하고 결정 경계에서 멀리 있음

**기존 Energy와의 차이**:
- Energy: 전체 클래스 logit 합산 → dominant class 정보 희석
- Margin: top-1과 top-2의 상대적 차이 → 결정 경계 인근 여부 명시적 측정

**주의**: Margin도 세 modality가 유사한 수준으로 학습되면 여전히 uniform 수렴 가능

### 4.1 [방향 D'] $E_{\text{uniform}}$ 보정 후 α 분포 확인

Adaptive하게 동작하는지 확인하는 가장 중요한 진단 지표:

**보정식**:
$$r_m^{\text{corrected}} = \sigma\!\bigl(-E_m + \ln C\bigr)$$

```python
# auxiliary_head_fusion_v2_7.py — _compute_confidence() energy 블록
energy = -torch.logsumexp(logits, dim=1)
num_classes = logits.shape[1]
energy_uniform = -torch.log(
    torch.tensor(num_classes, dtype=torch.float32, device=logits.device)
)
confidence = torch.sigmoid(-(energy - energy_uniform))  # = sigmoid(-energy + ln(C))
```

**확인 항목**:
```
1. per-sample std(α_m): 현재 0.008~0.010 → 목표 > 0.05
2. class별 평균 α_m: 클래스에 따라 dominant modality가 다른가?
3. Task별 α_m 변화: Task 0 vs Task 3에서 α 분포가 달라지는가?
4. Known vs Novel 샘플의 α 분포 차이
```

**예상 보정 후 α_m**:
- Task 0: (0.155, 0.449, 0.395) — Gyro/Acce 우세
- Task 3: (0.055, 0.472, 0.473) — RGB 거의 억제
- 현재: ≈ (0.330, 0.335, 0.340) — uniform

**성공/실패 판정**:
- per-sample std > 0.05 + OOD AUROC ≥ baseline 94.5% → 보정식만으로 충분
- per-sample std > 0.05이나 OOD AUROC 미개선 → 방향 A 필요
- per-sample std < 0.05 → IMU 크기 문제 지배적, 방향 A 직행

## 5. 권장 실험 순서

```
Step 1.   [진단] 현재 Ours의 α_m 분포 로깅 → 실제로 uniform 수렴인지 확인
              ✅ 완료: per-sample std = 0.008~0.010 (포화 확인됨)

Step 1.5. [방향 D' — E_uniform 보정] sigmoid 기준점 수정 후 α_m 재진단
              구현: confidence = sigmoid(-energy + ln(C))
              목표: per-sample std > 0.05 달성 여부 확인
              판단: std 개선 + AUROC 개선 → 충분 / 미개선 → Step 3로 진행
              소요: 코드 1줄 수정 + eval 재실행 (30분 이내)

Step 2.   [방향 D — τ 재탐색] E_uniform 보정 후에도 α 차이가 작은 경우
              τ 탐색 범위: {0.001, 0.005, 0.01, 0.05, 0.1, 0.5, 1.0}
              E_uniform 보정과 τ 탐색을 함께 수행하면 시너지 가능

Step 3.   [방향 A] R matrix 구축 + Class-Conditional α 구현 → 가장 유망한 방향
              Step 1.5 결과에 무관하게, class-agnostic 문제 완전 해결을 위해 필요
              Step 1.5에서 AUROC 개선 없으면 이 단계를 우선 진행

Step 4.   [방향 E] Top-2 Margin 기반 실험 → A 대비 추가 비교

Step 5.   [방향 B] Hybrid 실험 → A 구현 후 추가 이득 확인
```

## 6. 논문 내러티브 수정 방향

현재 서술: "energy score로 modality reliability를 추정"  
→ 실험 결과로 한계 드러남

개선 후 서술 옵션:
- **A 성공 시**: "class-conditional accuracy prior로 reliability를 측정하여 energy의 class-agnostic 한계를 극복"
- **Ablation table 재구성**: Adaptive-Energy → Adaptive-ClassConditional 로 대체 또는 추가

방향 A는 replay buffer를 이미 MoRST에서 활용하고 있으므로, **"MoRST와 MoAS의 결합이 더 긴밀해지는 narrative"**로 논문을 더 강화할 수 있다.
