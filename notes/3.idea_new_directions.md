# MoAS 재설계: Positive Synergy를 위한 새로운 방향

## 1. 기존 접근의 문제를 새로운 관점에서 재정의

### 1.1 기존 프레임: "어떤 modality를 얼마나 믿을 것인가"

현재 MoAS는 **modality-level scalar weight** `α_m`을 계산하여 모든 클래스 logit에 동일하게 적용한다:

```
z^MoAS_c = z_main,c + Σ_m α_m · z_{m,c}
```

이 구조에는 두 가지 설계 전제가 내재되어 있다:

1. **"Modality 단위 판단"**: modality m이 신뢰할 만하면 해당 modality의 **모든 클래스 logit**을 동등하게 반영
2. **"무조건 더하기"**: α_m > 0이면, z_{m,c}가 z_main에 도움이 되든 방해가 되든 무조건 합산

이 두 전제가 positive synergy를 구조적으로 불가능하게 만든다.

### 1.2 새로운 프레임: "z_main의 어떤 부분이 부족하고, modality가 그 부분을 보충할 수 있는가"

Positive synergy의 본질을 MaxLogit novelty detection 관점에서 정의하면:

| 샘플 유형 | 원하는 효과 | 의미 |
|---|---|---|
| **Known** | max logit **증가** | 올바른 클래스의 logit이 modality에 의해 강화됨 |
| **Novel** | max logit **유지 또는 감소** | 잘못된 확신이 추가되지 않음 |

즉, modality logit은 **z_main이 올바른 방향으로 갈 때만 밀어주고, 잘못된 방향으로 갈 때는 개입하지 않아야** 한다. 이것이 "selective reinforcement"이다.

### 1.3 현재 구조가 실패하는 구체적 시나리오

**시나리오 A: Known 샘플, z_main이 정확**
- z_main: class A에 높은 logit
- modality 1: class A에 높은 logit (도움)
- modality 2: class B에 높은 logit (방해)
- 현재: α_1 ≈ α_2 ≈ 1/3 이므로 **방해 신호도 동일하게 합산** → max logit 증폭 효과 감소

**시나리오 B: Novel 샘플**
- z_main: 전반적으로 낮은 logit (바람직)
- modality 1: class A에 착각하여 높은 logit (나쁜 신호)
- 현재: modality 1의 잘못된 확신이 **그대로 합산** → novel의 max logit 증가 → FPR95 악화

두 시나리오 모두에서 핵심 문제는 **modality logit이 z_main에 도움이 되는지 방해가 되는지를 판별하지 않고 무조건 합산**한다는 것이다.

## 2. 새로운 방향: Class-level Selective Reinforcement

### 2.1 핵심 아이디어

**Modality-level weight (α_m)를 폐기하고, class-level에서 선택적으로 modality 신호를 통합한다.**

질문을 바꾼다:
- 기존: "modality m을 얼마나 믿을 것인가?" → α_m (modality 단위 스칼라)
- 새로운: "**클래스 c에 대해** modality m의 logit이 z_main을 **보강하는가**?" → 클래스-modality 조합별 판단

### 2.2 방향 1: Agreement-Gated Integration

**원리**: z_main과 modality logit이 같은 클래스에 대해 **같은 방향**(둘 다 높거나 둘 다 낮음)일 때만 modality 기여를 허용한다.

```
δ_{m,c}(x) = z_{m,c}(x) · z_{main,c}(x)

        ┌ z_{m,c}(x)    if δ_{m,c} > 0   (같은 방향: 보강)
g_{m,c} = │
        └ 0              if δ_{m,c} ≤ 0   (반대 방향: 차단)

z^MoAS_c = z_main,c + (1/|M|) · Σ_m g_{m,c}
```

**동작 메커니즘**:

| 상황 | z_main,c | z_{m,c} | δ | g_{m,c} | 효과 |
|---|---|---|---|---|---|
| Known, 정답 클래스, modality 동의 | 높음 (+) | 높음 (+) | + | z_{m,c} 통과 | max logit 강화 |
| Known, 비정답 클래스, modality 동의 | 낮음 (-) | 낮음 (-) | + | z_{m,c} 통과 | 비정답 logit도 낮아져 마진 확대 |
| Known, 정답 클래스, modality 불일치 | 높음 (+) | 낮음 (-) | - | 0 (차단) | 방해 신호 제거 |
| Novel, modality 착각 | 낮음 (-) | 높음 (+) | - | 0 (차단) | 잘못된 확신 차단 |

**Soft version** (hard gating 대신 smooth gating):

```
g_{m,c} = z_{m,c} · σ(δ_{m,c} / τ)
```

σ는 sigmoid, τ는 temperature. δ가 양수일수록 gate가 열리고, 음수일수록 닫힌다.

**장점**:
- Modality가 z_main을 방해하는 방향의 기여를 구조적으로 차단
- Known 샘플에서 max logit 증폭, Novel 샘플에서 잘못된 증폭 방지 → 분리도 향상
- 추가 학습 파라미터 없음 (inference-only 유지)
- α_m 필요 없음 → energy, sigmoid, reliability score 계산 전부 불필요

---

### 2.3 방향 2: Residual Complementarity (잔차 보완)

**원리**: modality logit에서 z_main이 이미 가지고 있는 정보를 빼고, **z_main에 없는 새로운 정보(residual)**만 추출하여 합산한다.

```
# Modality의 잔차(= main이 모르는 정보)
r_{m,c}(x) = z_{m,c}(x) - z_{main,c}(x)

# 잔차 중 positive한 것만 선택 (main보다 더 확신하는 부분)
r^+_{m,c} = max(r_{m,c}, 0)

# 합산
z^MoAS_c = z_main,c + β · (1/|M|) · Σ_m r^+_{m,c}
```

**동작 메커니즘**:

| 상황 | z_main,c | z_{m,c} | r_{m,c} | r^+_{m,c} | 효과 |
|---|---|---|---|---|---|
| Main 약함, modality 강함 | 낮음 | 높음 | + | + | **보충**: main이 놓친 evidence 추가 |
| Main 강함, modality 약함 | 높음 | 낮음 | - | 0 | **무시**: 이미 충분, 방해 차단 |
| Main 강함, modality도 강함 | 높음 | 높음 | ≈ 0 | ≈ 0 | **보존**: 이미 좋은 상태 유지 |
| Novel (둘 다 약함) | 낮음 | 낮음 | ≈ 0 | ≈ 0 | **보존**: 낮은 score 유지 |

**해석**: z_main이 이미 자신 있는 부분은 그대로 두고, modality만이 추가 evidence를 가지고 있는 부분(r > 0)만 보충한다. 이것이 "positive synergy"의 직접적 구현이다.

**β**: 잔차 기여의 전체 스케일을 조절하는 hyperparameter.

**장점**:
- "main이 부족한 부분을 modality가 보충"이라는 의도를 수식으로 직접 표현
- Novel 샘플에서 modality도 보통 약하므로 residual ≈ 0 → 안전
- Energy, sigmoid, softmax, reliability 등 모든 중간 계산 불필요

---

### 2.4 방향 3: Main-Uncertainty-Gated Integration

**원리**: z_main의 **불확실성이 높을 때만** modality logit을 통합한다. z_main이 이미 확신하는 경우에는 modality를 추가하지 않는다.

```
# z_main의 불확실성 측정
u(x) = 1 - (max_c z_{main,c}(x) - max2_c z_{main,c}(x)) / |max_c z_{main,c}(x)|

# 또는 단순하게 margin 기반
margin(x) = z_{main}^{(1)}(x) - z_{main}^{(2)}(x)    # top1 - top2 logit
u(x) = σ(-margin(x) / τ)                              # margin 작을수록 불확실 → u 높음

# 불확실성이 gate 역할
z^MoAS_c = z_main,c + u(x) · (1/|M|) · Σ_m z_{m,c}
```

**동작 메커니즘**:
- `u(x) ≈ 1` (높은 불확실성): z_main이 헷갈려할 때 → modality logit이 **full scale로 기여** → 보충
- `u(x) ≈ 0` (낮은 불확실성): z_main이 확신할 때 → modality logit이 **거의 기여 안 함** → 이미 좋은 예측 보존

**Novelty detection 관점에서의 효과**:

| 샘플 유형 | Main의 상태 | u(x) | 결과 |
|---|---|---|---|
| Known (쉬운) | 확신 높음, margin 큼 | ≈ 0 | z^MoAS ≈ z_main (이미 높은 score 보존) |
| Known (어려운) | 불확실, margin 작음 | ≈ 1 | modality가 보충 → score 증가 |
| Novel | 불확실, margin 작음 | ≈ 1 | modality도 불확실 → 합산해도 score 낮음 유지 |

**주의**: Novel 샘플에서 u(x)가 높아 gate가 열리지만, modality logit 자체도 낮으므로 max logit이 크게 증가하지 않는다는 것이 기본 가정이다. 만약 특정 modality가 novel에도 높은 logit을 내면 문제가 될 수 있으므로, **방향 1 (agreement gating)과 결합**하는 것이 바람직하다.

## 3. 방향 간 결합: Hybrid 설계

세 방향은 상호 보완적이며, 결합하여 더 강력한 메커니즘을 구성할 수 있다.

### 3.1 방향 1 + 3: Uncertainty-Gated Agreement Integration

```
# Main의 불확실성
margin(x) = z_{main}^{(1)} - z_{main}^{(2)}
u(x) = σ(-margin(x) / τ_u)

# 클래스별 agreement gate
δ_{m,c} = z_{m,c} · z_{main,c}
a_{m,c} = σ(δ_{m,c} / τ_a)

# 결합: 불확실할 때만, 동의하는 방향으로만 기여
z^MoAS_c = z_main,c + u(x) · (1/|M|) · Σ_m a_{m,c} · z_{m,c}
```

**해석**: z_main이 불확실할 때(u ≈ 1), z_main과 같은 방향을 가리키는(a ≈ 1) modality logit만 합산한다.

### 3.2 방향 2 + 1: Agreement-Filtered Residual

```
# 잔차
r_{m,c} = z_{m,c} - z_{main,c}

# Agreement gate
a_{m,c} = σ(z_{m,c} · z_{main,c} / τ)

# 양의 잔차 + agreement 필터
z^MoAS_c = z_main,c + β · (1/|M|) · Σ_m a_{m,c} · max(r_{m,c}, 0)
```

**해석**: main보다 더 확신하는 부분(r > 0)이면서, main과 같은 방향인(a ≈ 1) 경우에만 보충한다.

## 4. 비교 요약

| 속성 | 기존 MoAS | 방향 1 (Agreement) | 방향 2 (Residual) | 방향 3 (Uncertainty) |
|---|---|---|---|---|
| Weighting 단위 | Modality-level (α_m) | Class-level (g_{m,c}) | Class-level (r^+_{m,c}) | Global scalar (u(x)) |
| 정보 활용 | Energy (magnitude) | z_main × z_m 부호 관계 | z_m - z_main 차이 | z_main의 margin |
| Negative 신호 처리 | 차단 안 함 | 차단 | 차단 | 차단 안 함 (결합 필요) |
| Known score 증폭 | 약함 (≈ uniform) | 동의 시 증폭 | main 약한 부분만 보충 | 불확실할 때만 증폭 |
| Novel score 보호 | 없음 | 불일치 차단 | 잔차 ≈ 0 | modality도 약함 (간접) |
| 추가 파라미터 | τ (temperature) | τ (soft gating용) | β (scale) | τ (margin sensitivity) |
| Energy/Sigmoid 필요 | 필요 | 불필요 | 불필요 | 불필요 |
| Inference-only | O | O | O | O |

### Phase 1: 개별 방향 검증

| 순위 | 실험 | 근거 |
|---|---|---|

| 2 | **방향 2: Residual Complementarity** | "부족한 부분 보충"이라는 동기와 가장 직접 대응. 방향 1과 성격이 다르므로 비교 가치 높음. |
| 3 | **방향 3: Uncertainty-Gated** | 단독으로는 novel 보호가 약하지만, 방향 1과의 결합용 기반. |

| 실험 | 구성 |
|---|---|
| Agreement + Uncertainty | 방향 1 + 3 결합 |

## 6. 구현 참고사항

### 6.1 기존 코드 변경 최소화

기존 MoAS 파이프라인에서 **energy, sigmoid, softmax(α) 계산을 제거**하고, 해당 부분을 새 gating 로직으로 교체하면 된다.

기존:
```python
# 제거할 부분
E_m = -torch.logsumexp(z_m, dim=-1)
r_m = torch.sigmoid(-E_m)
alpha = F.softmax(r / tau, dim=0)
z_moas = z_main + sum(alpha[m] * z_m[m] for m in modalities)
```

방향 1 (Agreement-Gated, soft version):
```python
# 교체할 부분
z_moas = z_main.clone()
for m in modalities:
    delta = z_m[m] * z_main          # class-wise agreement (element-wise)
    gate = torch.sigmoid(delta / tau) # soft gate
    z_moas += (1 / len(modalities)) * gate * z_m[m]
s = z_moas.max(dim=-1).values
```

방향 2 (Residual Complementarity):
```python
# 교체할 부분
z_moas = z_main.clone()
for m in modalities:
    residual = z_m[m] - z_main                 # 잔차
    positive_residual = torch.clamp(residual, min=0)  # 양의 잔차만
    z_moas += (beta / len(modalities)) * positive_residual
s = z_moas.max(dim=-1).values
```

방향 3 (Uncertainty-Gated):
```python
# 교체할 부분
topk = z_main.topk(2, dim=-1).values
margin = topk[:, 0] - topk[:, 1]              # top1 - top2
u = torch.sigmoid(-margin / tau).unsqueeze(-1) # uncertainty gate

z_moas = z_main.clone()
for m in modalities:
    z_moas += u * (1 / len(modalities)) * z_m[m]
s = z_moas.max(dim=-1).values
```

### 6.2 Hyperparameter 탐색 범위

| 방향 | Parameter | 제안 범위 |
|---|---|---|
| 방향 1 | τ (agreement sensitivity) | {0.1, 0.5, 1.0, 5.0, 10.0} |
| 방향 2 | β (residual scale) | {0.1, 0.3, 0.5, 1.0} |
| 방향 3 | τ (margin sensitivity) | {0.5, 1.0, 2.0, 5.0} |
