# Step 1: α_m 분포 로깅 진단 구현 명세

## 목적

> "현재 Ours(Adaptive Reliability)의 α_m 분포가 실제로 uniform(1/3, 1/3, 1/3)에 수렴하는가"를 실험적으로 확인한다.

ablation 결과(Ours ≈ Uniform Average)가 **α_m 포화**에 의한 것인지, 아니면 **모델 자체의 특성**에 의한 것인지를 구분하는 것이 목표다.

---

## 1. 현재 α_m 생성 경로 파악

### 1.1 사용 파일 확인

- **fusion 모듈**: `models/fusion/auxiliary_head_fusion_v2_7.py` (`exp_mmea_tbn_mmeader.json`: `"fusion_type": "auxiliary_head_v2_7"`)
- **OOD 평가**: `ood/methods/unified_ood_detector.py` (`"ood_methods": ["MaxLogit_Hybrid_ConfNormalized", ...]`)

### 1.2 코드 흐름 추적

```
[학습/평가 시 — auxiliary_head_fusion_v2_7.py]
  forward()
    ├─ _compute_confidence(aux_logits)    ← confidence_method="energy" 기준
    │    energy = -logsumexp(logits)      ← E_m = -log Σ_c exp(z_{m,c})
    │    confidence = sigmoid(-energy)    ← r_m = σ(logsumexp)  ← ★ 포화 지점
    │
    └─ weights = torch.ones_like(...)     ← ★ v2_7 핵심: confidence는 반환만 하고
                                               가중치에는 전혀 사용 안 함! (항상 1:1:1)
       → confidences dict는 return 값에만 담겨서 전달됨

[OOD 평가 시 — mmeabase.py evaluate_ood()]
  _collect_outputs(test_loader)
    → outputs['confidences'] = {RGB: tensor[N], Gyro: tensor[N], Acce: tensor[N]}
       (각 값 = sigmoid(-energy), 이미 포화된 상태로 수집됨)

[OOD score 계산 — unified_ood_detector.py]
  _compute_hybrid_scores()
    └─ _fuse_logits_conf_normalized()
         conf_stacked = stack(confidences)          # [3, N]  ← 포화된 r_m 값
         normalized_weights = softmax(conf_stacked, dim=0)  # ← α_m
         # r_RGB ≈ r_Gyro ≈ r_Acce ≈ 0 이면 → softmax([0,0,0]) = (1/3, 1/3, 1/3)
         fused_logits = z_main + Σ α_m · z_m
```

> **v2_7의 특수성**: v2_7은 fusion 시 confidence를 **가중치로 쓰지 않는다**.
> confidence는 오직 `return dict`의 `'confidences'` 키로만 전달되며,
> OOD 평가 시 `_fuse_logits_conf_normalized()`에서 비로소 α_m 계산에 사용된다.
> 따라서 학습 중 fusion 가중치와 OOD scoring 가중치는 완전히 분리된 경로다.

### 1.3 확인이 필요한 값들

| 확인 대상 | 예상 값 (포화 시) | 위치 |
|---|---|---|
| `energy` (raw) | 큰 양수 (e.g., 10~30) | `_compute_confidence()` 내부 |
| `confidence` = `sigmoid(-energy)` | ≈ 0.0 (포화) | `_compute_confidence()` 반환값 |
| `normalized_weights` (= α_m) | ≈ (0.33, 0.33, 0.33) | `_fuse_logits_conf_normalized()` 내부 |
| modality 간 confidence 차이 `std` | ≈ 0 | `_collect_outputs()` 결과 |

### 3.2 `ood/methods/unified_ood_detector.py` — `_fuse_logits_conf_normalized()` 수정

**목적**: 실제 modality weight(=α_m)의 통계와 분포를 로깅하여 uniform 수렴 여부 확인

**수정 위치**: `normalized_weights = torch.softmax(conf_stacked, dim=0)` 직후

```python
# Softmax로 confidence 정규화 (dim=0: 모달리티 차원에서 합=1)
normalized_weights = torch.softmax(conf_stacked, dim=0)  # [num_modalities, batch]

# ─── [진단 로깅 추가] ────────────────────────────────────────────
# normalized_weights shape: [M, B]  (M=모달리티 수, B=배치 크기)
# 1회만 출력하기 위한 모듈 레벨 플래그 사용
if not getattr(self, '_alpha_logged', False):
    w_np = normalized_weights.detach().cpu().numpy()  # [M, B]
    modality_names = ['RGB', 'Gyro', 'Acce']

    logging.info("=" * 60)
    logging.info("[α-Diag] Modality Weight (α_m) Distribution")
    logging.info(f"  shape: {w_np.shape}  (modalities × batch)")

    for i, mod in enumerate(modality_names[:w_np.shape[0]]):
        w_i = w_np[i]  # [B]
        logging.info(
            f"  α_{mod}: mean={w_i.mean():.4f}, std={w_i.std():.4f}, "
            f"min={w_i.min():.4f}, max={w_i.max():.4f}"
        )

    # 모달리티 간 분산 (클수록 adaptive하게 동작)
    per_sample_std = w_np.std(axis=0)   # [B]: 각 샘플에서 3개 모달리티 weight의 std
    logging.info(
        f"  per-sample std(α_RGB, α_Gyro, α_Acce): "
        f"mean={per_sample_std.mean():.4f}, max={per_sample_std.max():.4f}"
    )
    logging.info(
        f"  → uniform 기준: std ≈ 0.0000  |  "
        f"완전 one-hot 기준: std ≈ {(2/3)**0.5 / 3:.4f}"
    )
    logging.info("=" * 60)
    self._alpha_logged = True
# ──────────────────────────────────────────────────────────────────
```

## 4. 진단 결과 해석 기준

| per-sample std(α) | 해석 |
|---|---|
| ≈ 0.0000 | 완전 uniform → adaptive 미동작 (포화 확인) |
| 0.001 ~ 0.05 | 거의 uniform → 실질적 adaptive 효과 없음 |
| 0.05 ~ 0.15 | 어느 정도 adaptive하게 동작 중 |
| > 0.15 | 충분히 adaptive (one-hot 방향) |

> 참고: 3개 모달리티 완전 one-hot 기준 std ≈ 0.272

## 8. v2_7 고유 주의사항

v2_7은 fusion 시 confidence를 가중치로 사용하지 않는다는 점에서 v2_8과 근본적으로 다르다.

| 구분 | v2_7 (현재 사용) | v2_8 |
|---|---|---|
| 학습 시 fusion 가중치 | 항상 1:1:1 균등 | raw confidence 값 사용 |
| confidence 계산 목적 | OOD return용 + 분석용 | fusion 가중치 + OOD return용 |
| α_m 포화의 영향 | **OOD scoring 경로에만** 영향 | 학습 fusion에도 영향 |

따라서 v2_7에서 α_m 포화는 **OOD score 계산에만** 영향을 미치며, 학습된 modality representation 자체의 품질과는 별개의 문제다. 진단을 통해 포화 여부를 확인한 뒤, 방향 A(Class-Conditional)로 OOD scoring 경로만 교체하면 된다.

## 9. 추가 진단: Modality별 Energy Scale 분석 및 $E_{\text{uniform}}$ 보정

> **이 섹션은 Step 1 실험 완료 후 발견된 추가 진단 과정이다.**  
> α_m 포화가 확인된 뒤, "포화의 구체적 원인이 sigmoid의 잘못된 기준점 때문인가?"를 검증한다.

---

### 9.1 추가 진단의 동기

Step 1 진단에서 `confidence = sigmoid(-energy)` 값이 ~0.87~0.99로 포화됨을 확인하였다.  
그런데 energy는 **음수** 값으로 나왔다 (mean ≈ -2.73). 이는 기존 예상(큰 양수)과 다른 결과다.

> **핵심 의문**: `sigmoid(-(-2.73)) = sigmoid(2.73) ≈ 0.94` → 이미 포화.  
> 그러나 왜 -2.73이 포화를 유발하는가? $C$개 균등분포의 이론적 energy는?

$$E_{\text{uniform}} = -\log\sum_c \frac{1}{C} = -\log(C \cdot \frac{1}{C}) = -\log 1 \neq 0 \quad\text{(틀린 직관)}$$

올바른 계산:
$$E_{\text{uniform}} = -\log\sum_c \exp\left(\log\frac{1}{C}\right) = -\log\left(C \cdot \frac{1}{C}\right) = -\log 1 = 0 \quad\text{(uniform logit=0일 때)}$$

그러나 실제로 모든 logit이 0이 아니라 **softmax 균등 분포를 만드는 logit**일 때:
$$z_c = \text{const} \Rightarrow \log\sum_c \exp(z_c) = \log(C \cdot e^{z}) = z + \log C$$
$$E_{\text{uniform}} = -(z + \log C) \approx -\log C \quad\text{(z=0 기준)}$$

따라서 **균등 분포 가정 시 참조 기준값**은:
$$\boxed{E_{\text{uniform}} = -\ln C}$$

| C (클래스 수) | $E_{\text{uniform}} = -\ln C$ |
|---|---|
| 8  | -2.079 |
| 16 | -2.773 |
| 24 | -3.178 |
| 32 | -3.466 |

---

### 9.2 추가 로깅 구현

기존 `_compute_confidence()` 로깅에 다음 내용을 추가한다:

**수정 파일**: `models/fusion/auxiliary_head_fusion_v2_7.py`  
**수정 위치**: `_compute_confidence()` 내부 `if not getattr(self, f'_energy_logged_{modality_name}', False):` 블록

```python
elif self.confidence_method == "energy":
    energy = -torch.logsumexp(logits, dim=1)
    confidence = torch.sigmoid(-energy)

    if not getattr(self, f'_energy_logged_{modality_name}', False):
        e_np = energy.detach().cpu().numpy()
        c_np = confidence.detach().cpu().numpy()
        num_classes = logits.shape[1]

        # ─── E_uniform 기준값 계산 ────────────────────────────
        import math
        e_uniform = -math.log(num_classes)
        sigmoid_at_uniform = 1 / (1 + math.exp(e_uniform))  # = sigmoid(-E_uniform)

        # 포화 비율: E < E_uniform → sigmoid(-E) > sigmoid(-E_uniform) → 과확신
        sat_ratio = (e_np < e_uniform).mean() * 100

        logging.info(
            f"[α-Diag][{modality_name}] energy(raw):  "
            f"mean={e_np.mean():.3f}, std={e_np.std():.3f}, "
            f"min={e_np.min():.3f}, max={e_np.max():.3f}  "
            f"(C={num_classes}, E_uniform={e_uniform:.3f})"
        )
        logging.info(
            f"[α-Diag][{modality_name}] sigmoid(-E):  "
            f"mean={c_np.mean():.4f}, std={c_np.std():.4f}, "
            f"min={c_np.min():.4f}, max={c_np.max():.4f}  "
            f"(sigmoid at E_uniform={sigmoid_at_uniform:.4f})"
        )
        logging.info(
            f"[α-Diag][{modality_name}] 포화 진단:  "
            f"E < E_uniform({e_uniform:.2f}) 인 샘플 = {sat_ratio:.1f}%  "
            f"→ 이 비율이 높을수록 confidence 포화 심각"
        )
        setattr(self, f'_energy_logged_{modality_name}', True)
```

**변경 포인트**:
- `_energy_logged` 단일 플래그 → `_energy_logged_{modality_name}` per-modality 플래그로 교체
- `_compute_confidence(self, logits)` → `_compute_confidence(self, logits, modality_name=None)` 시그니처 변경
- `update_task()` 리셋 루프도 per-modality 방식으로 변경:

```python
def update_task(self, task_id):
    # ... 기존 코드 ...
    for mod in getattr(self, 'modality', ['RGB', 'Gyro', 'Acce']):
        setattr(self, f'_energy_logged_{mod}', False)   # ← 수정: per-modality 리셋
```

### 9.4 설계 결함 진단

$$r_m = \sigma(-E_m) \qquad\text{(현재 구현 — 설계 결함)}$$

**문제**: sigmoid 0.5 기준점이 $E_m = 0$으로 고정  
**올바른 0점**: $E_{\text{uniform}} = -\ln C$ (클래스 수에 따라 달라져야 함)

$$\boxed{r_m^{\text{corrected}} = \sigma\!\bigl(-(E_m - E_{\text{uniform}})\bigr) = \sigma(-E_m + \ln C)}$$

| 상태 | $E_m$ 위치 | 현재 $\sigma(-E_m)$ | 보정 후 $\sigma(-E_m + \ln C)$ |
|------|-----------|---------------------|-------------------------------|
| 완전 균등 분포 | $E_m = E_{\text{uniform}}$ | $\sigma(\ln C) > 0.5$ (**편향**) | **0.5** (중립) |
| ID (확신) | $E_m \ll E_{\text{uniform}}$ | ≈ 1.0 (포화) | > 0.5 (적절) |
| OOD (불확실) | $E_m \gg E_{\text{uniform}}$ | 낮음 | < 0.5 (적절) |

## 10. Step 1.6 — Z-score Energy Normalization 진단 명세

> **동기**: Step 1.5의 $E_{\text{uniform}}$ 보정은 sigmoid 절대 기준 문제를 해결했으나,  
> IMU logit magnitude가 RGB보다 구조적으로 2.5~3.0 단위 크다는 cross-modality scale 차이는 해결하지 못했다.  
> Z-score 정규화로 modality 간 energy 스케일을 통일하여 이 문제를 해결한다.

---

### 10.1 보정식

$$z_m = \frac{E_m - \mu_{m,\text{mem}}}{\sigma_{m,\text{mem}}}$$

$$\text{confidence}_m = -z_m$$

- $\mu_{m,\text{mem}}$, $\sigma_{m,\text{mem}}$: modality $m$의 memory buffer 데이터에서 계산한 energy 평균 및 표준편차
- $z < 0$ (에너지가 memory 평균보다 낮음, 확신 높음): confidence > 0 
- $z = 0$ (에너지 = memory 평균, 중립): confidence = 0
- $z > 0$ (에너지가 memory 평균보다 높음, 불확실): confidence < 0

> **Memory buffer 사용 근거**: Train loader는 current task 데이터만 포함하므로,  
> continual learning 환경에서 이전 task의 energy 분포를 반영하지 못한다.  
> Memory buffer (exemplar set)는 모든 학습된 클래스의 대표 샘플을 포함하므로,  
> 전체 학습 분포에 대한 더 정확한 energy 통계를 제공한다.

> **NOTE**: 초기 설계에서는 `confidence = sigmoid(-z)` 를 사용했으나,  
> softmax α_m 계산 전에 sigmoid를 거치면 스케일이 다시 압축되므로,  
> raw `-z` 값을 직접 softmax 입력으로 사용하는 것이 분별력을 극대화한다.

### 10.2 Step 1.5와의 차이

| 항목 | Step 1.5 ($E_{\text{uniform}}$ 보정) | Step 1.6 (Z-score) |
|------|-------------------------------------|--------------------|
| 기준점 | $E_{\text{uniform}} = -\ln C$ (이론적 균등분포) | $\mu_{m,\text{mem}}$ (경험적 memory 평균) |
| 스케일링 | 없음 (raw energy 차분) | $\sigma_{m,\text{mem}}$ 으로 나눔 |
| Cross-modality scale | ✗ 해결 못함 (IMU magnitude 그대로) | ✅ 해결 (modality별 정규화) |
| 외부 통계 필요 | 없음 (C만 필요) | ✅ 필요 (memory buffer forward pass) |
| 이론적 의미 | "균등분포가 불확실(0.5)" | "memory 평균이 중립(0)" |

### 10.3 구현 명세

**`auxiliary_head_fusion_v2_7.py`**:
- `__init__`: `self._energy_stats = {}` 추가
- `set_energy_stats(energy_stats)`: 외부에서 `{modality: (mean, std)}` 주입
- `_compute_confidence()` energy 블록:
  - `_energy_stats[modality]` 존재 시 → Z-score (`norm=zscore`)
  - 없으면 → fallback (e_uniform 또는 raw)

**`mmeabase.py`**:
- `_compute_energy_stats_from_memory(data_manager)`: memory loader로 modality별 energy 수집 → stats 계산
- `inference_mode_evaluation()`: checkpoint load → memory build → energy stats 계산 → evaluate

### 10.4 `energy_norm_method` 선택 파라미터

JSON config 에서 `energy_norm_method` 를 선택할 수 있도록 파이프라인에 파라미터를 추가한다:

```json
{
  "energy_norm_method": "zscore"   // "zscore" | "e_uniform" | "raw"
}
```

전달 경로: `exps/JSON` → `baseline_tbn.py` → `fusion/__init__.py` `get_fusion()` → `AuxiliaryHeadFusionV2_7.__init__`

| 옵션 | 수식 | 외부 통계 | 의미 |
|------|------|----------|------|
| `"zscore"` | $\text{conf} = -z = -(E - \mu_{\text{mem}})/\sigma_{\text{mem}}$ | ✅ 필요 (memory buffer) | 상대적 비전형성 (relative atypicality) |
| `"e_uniform"` | $\text{conf} = \sigma(-(E - E_{\text{uniform}}))$ | 없음 | 균등분포 기준 절대 신뢰도 |
| `"raw"` | $\text{conf} = \sigma(-E)$ | 없음 | 원시 energy sigmoid (baseline) |
